The Business Problem

Personal finance apps struggle to accurately categorize transactions, leading to unreliable spending insights and poor user trust. This project tackles that problem by building a machine learning classifier trained on behavioral transaction data, moving beyond rule-based categorization toward a model that learns spending patterns automatically.

Approach and Methodology

  • Exploratory Data Analysis: Uncovered 3 behavioral spending insights by analyzing patterns across 7 days of the week and 4 time-of-day slices.
  • Feature Engineering: Engineered temporal features including month, day, day of week, time slice, and weekend flag to capture spending rhythm.
  • Ablation Study: Evaluated 4 models (XGBoost, Random Forest, Decision Tree, Logistic Regression) across 4 dataset configurations to isolate the best feature and encoding combination.
  • Model Selection: Selected XGBoost on signed ordinal encoding with 81% single-split accuracy and 74% K-Fold accuracy, prioritizing generalization over overfitting.
  • Cross Validation: Used StratifiedKFold to ensure reliable evaluation across all 5 spending categories: Food, Household, Other, Salary, and Transportation.

System Architecture

  • Model Layer: XGBoost classifier serialized with joblib, served via a model.py module with clean prediction logic.
  • API Layer: FastAPI app with a /predict endpoint, Pydantic schema for request validation, and automatic Swagger UI documentation.
  • Containerization: Dockerized with a pinned requirements.txt for fully reproducible deployments across environments.
  • Cloud Deployment: Deployed to Render via GitHub, with the Docker container serving live predictions via a public URL.
  • Agentic AI Layer: Groq LLM acts as the reasoning brain, extracting structured transaction features from plain English input and returning spending category predictions with insights.
  • Frontend: Streamlit app with a premium dark UI, deployed on Streamlit Community Cloud for public access.

Tools and Technologies

Python, XGBoost, scikit-learn, FastAPI, Uvicorn, Docker, Render, Groq LLM, Streamlit, pandas, joblib, python-dotenv

Key Results

  • 81% classification accuracy across 5 spending categories on single split evaluation.
  • 74% K-Fold accuracy confirming model generalization, not overfitting.
  • Sub-second prediction response times on the live deployed API.
  • 3 behavioral insights surfaced through EDA informing feature design decisions.
  • Natural language to prediction pipeline working end to end via Groq LLM agent.
  • Live Streamlit frontend deployed publicly for anyone to try.
