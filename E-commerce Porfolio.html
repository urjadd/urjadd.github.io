E-commerce Sales Dashboard (Flagship)

A complete sales analytics case study involving schema design, SQL automation, dashboarding, and segmentation. This project simulates a business intelligence pipeline for a retail company.

🔍 Problem Statement

How can we enable business stakeholders to monitor product and country-level sales performance, and uncover hidden insights to optimize customer strategies?

🚧 Problems I Faced & How I Solved Them

  • Dirty CSVs: Column mismatches, nulls, data type inconsistencies.
    Fix: Cleaned using Pandas, standardized schema with strict dtype enforcement.
  • Manual data loading pain: Needed fast iterative development.
    Fix: Wrote a custom SQL insert generator to convert CSVs into ready-to-run SQL scripts.
  • Multiple disconnected files: Schema, inserts, queries scattered.
    Fix: Modularized code with structured folders and a master Python runner (`main.py`).
  • Low clarity in business output: Queries were too technical.
    Fix: Built a Tableau dashboard with clean KPI summaries and segment-level visualizations.

🛠️ Tools & Technologies

  • Python (Pandas, sqlite3, automation scripts)
  • SQL (Schema design, business analytics)
  • Tableau (.twb workbook, visual storytelling)
  • Jupyter Notebook (data inspection, boxplots)

⚙️ Project Flow

  • Cleaned and validated raw CSVs in Python
  • Created SQL schema with referential integrity
  • Automated data inserts with custom generator
  • Built 10+ SQL queries to derive key insights
  • Designed Tableau dashboard for business use

📊 Key Insights

  • Top 5 products contributed over 50% of total revenue
  • 3 countries drove most of the order volume
  • “Medium spenders” segment showed highest repeat rate
  • Identified products never purchased — potential dead stock
